## 🔄 工作流程

### 1️⃣ 写入需求

在语雀文档写入工作需求：
- 任务描述
- 目标输出
- 截止时间
- 优先级

### 2️⃣ AI 分工

| 任务类型 | 执行者 | 说明 |
|---------|--------|------|
| 文档处理 | Kimi | 读写语雀、整理文档 |
| 本地执行 | OpenClaw Mac | 运行脚本、操作文件 |
| 后台任务 | OpenClaw WSL2 | 定时任务、API 调用 |
| 通信通知 | OpenClaw | 飞书/Telegram 消息 |

### 3️⃣ 执行反馈

- Kimi 更新语雀进度
- OpenClaw 推送 GitHub 提交
- 飞书通知完成状态

## 📝 使用示例

### 示例 1: 设计咖啡店菜单

**语雀写入:**
```
任务：设计春季新品菜单
要求：
- 3 款咖啡 + 2 款甜点
- 包含朝文/中文/英文
- 符合朝鲜族特色
输出：PDF 菜单文件
```

**执行流程:**
1. Kimi 读取需求，调研市场
2. OpenClaw 生成设计方案
3. Kimi 整理文档，输出 PDF
4. 飞书通知完成

---

### 示例 2: 自动化客户回复

**语雀写入:**
```
任务：创建自动回复脚本
功能：
- 识别常见问题
- 自动回复营业时间/位置
- 转接人工客服
输出：可部署的脚本
```

**执行流程:**
1. OpenClaw 编写脚本
2. Kimi 测试并优化
3. 部署到 Mac 分身
4. 语雀记录使用说明

## 📋 任务模板

### 在语雀写入任务时使用这个格式：

```markdown
## 任务：[任务名称]

**描述:** [详细说明]

**要求:**
- [要求 1]
- [要求 2]
- [要求 3]

**输出:** [期望的输出格式]

**截止时间:** [日期]

**优先级:** 高/中/低
```

## 🎯 最佳实践

1. **任务明确** - 描述清晰，避免歧义
2. **分工合理** - 让合适的 AI 做合适的事
3. **及时反馈** - 完成后更新语雀状态
4. **版本管理** - 重要文件用 GitHub 管理
